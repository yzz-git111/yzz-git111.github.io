面向对象思想

一个简单的实例

//定义了一个Fish类
class Fish {
   int weight = 1;
}

//定义了一个Lake类
class Lake {
    //Lake类中 有一个Fish类的对象fish作为类的属性
   Fish fish;
   void setFish(Fish s){
       fish = s;
   }
   void foodFish(int m) {
      fish.weight=fish.weight+m;
   }
}
public class E {
   public static void main(String args[]) {
       Fish  redFish = new Fish();
       System.out.println(redFish.weight);  //【代码1】 答案是1
       Lake lake = new Lake();
       lake.setFish(redFish);
       lake.foodFish(120);
       System.out.println(redFish.weight);  //【代码2】答案是121
       System.out.println(lake.fish.weight);  //【代码3】答案是121  (redFish和fish指向同一块内存)
    }
}

这段代码实现的是在一个池塘里面喂鱼的功能。
采用面向过程来写的话，大概率会实现两个函数void setFish(),void foodFish().创建一个变量或者结构体代表🐟，然后调用这两个函数实现养鱼并且给它喂食使🐟的体重增长。

而面向对象思想：把鱼抽象成为一个类，同时池塘也抽象成为一个类。鱼这个类只有鱼自身的属性weight。而养鱼和给鱼喂食由于都是在池塘里进行的，可以将这两个方法在Lake类中实现，只需要在Lake类有一个Fish类的对象作为属性(池塘中有鱼)。调用时，需要先new一个Lake的对象(先有个池塘),new一个Fish的对象(先要有鱼)。然后调用Lake的方法setFish(),把new的Fish的对象传递进去，意味着将鱼放到池塘中喂养。然后喂食时调用Lake的foodFish()方法，传递一个Fish的对象，意味着对池塘中的某个鱼进行喂食。

个人觉得面向对象需要将所有的东西都抽象成为一个类，结合实际来编程。目前理解还不是很深。

面向对象语言的三大特性：封装、继承、多态。java作为一门面向对象的语言，将一切都抽象为 类，实现万物皆对象。1

封装

封装的好处：

* 提高数据的安全性。
* 隐藏代码实现的细节
* 便于维护代码
  1. 使属性私有化，并且通过get(),set()方法来操作。

实例：

class Student
{
    private String id;  
    private int age;

    Student()
    {
    }
    public int getAge()
    {
        return age;
    }

    public void setAge(int age)
    {
        if(age<0 || age>25)
        {
            System.out.println("年龄不合法！");
            return;
        }
        this.age = age;
    }

    public String getId(String password)
    {
        if(!password.equals("123456"))
            return "没有权限!";
        return id;
    }

    public void setId(String id)
    {
        this.id = id;
    }
}

对于上述的实例，在类中定义了两个成员age,id。为了保护隐私，将他们都设置为private。然后通过统一的set和get接口来操作这两个属性。

对于setAge方法，假设age不能小于0或者大于25，那么在set时就可以进行一下判断是否合法，对于不合法的数值，就输出一个提示。合法才允许设定。

同样的，对于getId方法，为了保护学生的id不泄露，必须特定的(假设有密码，密码通过才表示有权限访问)条件才能有效访问，那么可以在get方法传递一个参数(假定就是密码)来进行判断。

通过简单的操作能够适当的保护数据的安全性。

同时，对于他人，只知道输入有效数值才能够通过setAge()，通过输入密码才能getId(),但是这些操作的实现对于他们是透明的。隐藏了具体实现的细节。

同时，统一的set(),get()接口便于代码的维护。提高可读性。

  1. 对于类的封装
    当我们需要多次重复的使用一个类，而类中有很多属性是重复的，这时候我们可以将类的各种属性都设定，将这个类封装成新的一个类，直接new这个类。这样做可以实现代码的复用。

继承

假定已经实现了一个类A，而类B在A的基础上进行了增加，如果不使用继承，那么需要在B类重新写一下已经在A类写好的方法。而使用了继承之后，对于A类中已经实现的方法B默认拥有了。

  • 所以继承最大的优点就是：实现代码的可复用性。

实例：假设将小学生定义了一个类PrimaryStudent，小学生会加法和减法，即在里面实现了方法add()，和sub()。现在又要定义一个高中生类HighschoolStudent,高中生会乘法和除法，除此之外高中生也一定会乘法和除法。当HighschoolStudent继承PrimaryStudent只需要实现方法multiple()和divi().

  • 子类继承父类的所有非private的对象和方法
  • 子类重写父类的方法后，调用super.调用被子类隐藏的方法和属性。
  • 子类调用父类构造方法时，要在子类构造方法的第一行调用super()。

java中所有的类都继承了Object类。而Object类中实现了getClass(),toString(),hashCode()等方法。所以每个类都有这些方法。

java只支持单继承。

多态

多态即同一方法根据不同的对象而采用不同的实现。
多态存在的条件：

  • 有继承关系
  • 子类重写父类关系
  • 父类的引用指向子类(子类类型的引用赋值给父类)
  1. 最基础的，以下面代码为例，son继承了father类。father定义了say方法。son的引用s1指向son类和father的引用f2指向son类，say方法都是输出father。
    虽然f2是father的引用，但是son也继承且为重写father的say，所以子类也默认有say方法。
class father
{
    public void say()
    {
        System.out.println("father");
    }
}

class son extends father{}

public static void main(String[] args)
{
    son s1 = new son();
    s1.say();   //father
    father f2 = new son();
    f2.say();   //father
}
  1. 在子类中重写父类的方法。那么会隐藏父类的方法，执行子类的方法。
    这时s1和f2调用的都是子类的say方法，输出son。
class son extends father
{
    public void say()
    {
        System.out.println("son");
    }
}
  1. 在子类新实现一个方法eat()。由于这是子类独有的方法，所以父类的引用f2不能调用eat()。
    想要调用eat()，需要将f2强转为子类.
class son extends father
{
    public void say()
    {
        System.out.println("son");
    }
    public void eat()
    {
        System.out.println("eat");
    }
}

public static void main(String[] args)
{
    son s1 = new son();
    father f2 = new son();

    s1.say();
    f2.say();

    s1.eat();
    //f2.eat();不能调用
    ((son)f2).eat(); //强转后才能调用。
}

总结：对象能够调用那些方法，主要看引用(左边)的类型。（f3不能调用eat）
当子类的引用指向子类，对象可以调用子类自己实现的方法和继承的方法。（s1调用eat）
当父类的引用指向子类，子类重写了父类的方法，对象调用的也是子类的方法。（f2.say输出son）

只有方法有多态，属性不存在多态。

抽象类和接口

  1. 相同点：
    抽象类和接口都是对于类的抽象。

都不能被实例化。

都可以包含方法的声明。

子类必须实现未实例化的方法。

  1. 不同点：
    一个子类只能继承一个抽象类；一个子类可以实现多个接口(变相的多继承)。

一个抽象类可以实现多接口；接口不能继承抽象类，只能继承多接口。

抽象类可以包括常量，变量，方法的声明，普通方法，构造方法；而接口只能包含全局变量和方法的声明。

接口只能定义声明；而抽象类可以包含方法声明和实现。接口比抽象类更抽象。

接口的常量默认为public static final，方法默认为public abstract；而抽象类可以有各种权限，且子类的权限不能低于父类的。

应优先考虑接口，避免单继承的局限性

上转型对象与接口回调

  1. 上转型对象
    父类 x(引用类型) = new 子类(); 那么x就是子类的上转型对象。
public class test
{
    public static void main(String[] args)
    {
        //animal1，animal2，animal3就是dog类的对象的上转型对象。
        animal animal1 = new dog(); //第一种实现

        animal animal2;     //第二种实现
        dog dog1 = new dog();
        animal2 = dog1;

        animal animal3;     //第三种实现
        animal3 = new dog();
    }
}

class animal
{
}

class dog extends animal
{
}

注意点：
上转型对象不可以操作子类新增的成员和方法(和多态的f3不能调用eat一样)。对象能够调用那些方法，主要看引用(左边)的类型。

可以将对象的上转型对象再强转成子类对象，这时该对象又具有了子类的所有属性和方法。(和多态的((son)f2).eat();一样)

如果子类重写了父类的方法，当对象的上转型对象调用这个方法时一定调用的是子类重写的方法(和多态的f2.say输出son一样)

可以父类 x(引用类型) = new 子类(); 但不可以子类 x(引用类型) = new 父类;

  1. 接口回调
    接口回调指的是：实现某一接口的类 创建的对象 的引用 赋值给 该接口声明的接口变量，那么该接口变量就可以调用被类实现的接口方法。

接口 x(引用类型) = new 实现接口的子类;

这里的理解和上转型对象很类似。

public class test
{
    public void print(merchandise m)    //接口声明的变量为接口变量
    {
        System.out.println("price="+m.price());
    }
    public static void main(String[] args)
    {
        test test = new test();
        test.print(new computer());     //接口回调  
        //即merchandise m = new computer(new的匿名类为接口的类传教的对象的引用，m为该接口声明的接口变量)
        test.print(new book());         //接口回调
    }
}

interface merchandise
{
    int price();
}

class computer implements merchandise
{
    private final int price = 10000;

    public int price()
    {
        return price;
    }
}

class book implements merchandise
{
    private final int price = 10;

    public int price()
    {
        return price;
    }
}

重写和重载

  1. 重写
    重写是子类对父类的允许访问的方法的实现过程进行重新编写, 返回值和形参都不能改变。
    重写的好处在于子类可以根据需要，定义特定于自己的行为。 也就是说子类能够根据需要实现父类的方法。
class Animal{
   public void move(){
      System.out.println("动物可以移动");
   }
}
 
class Dog extends Animal{
   public void move(){  //子类重写move()
      System.out.println("狗可以跑和走");
   }
}

重写的规则：

1.参数列表与被重写方法的参数列表必须完全相同。

2.访问权限不能比父类中被重写的方法的访问权限更低。例如：如果父类的一个方法被声明为 public，那么在子类中重写该方法就不能声明为 protected。

3.父类的成员方法只能被它的子类重写。

4.声明为 final 的方法不能被重写。

5.声明为 static 的方法不能被重写，但是能够被再次声明。

6.子类和父类在同一个包中，那么子类可以重写父类所有方法，除了声明为 private 和 final 的方法。

7.子类和父类不在同一个包中，那么子类只能够重写父类的声明为 public 和 protected 的非 final 方法。

8.构造方法不能被重写。

  1. 重载

重载(overloading) 是在一个类里面，方法名字相同，而参数不同。返回类型可以相同也可以不同。
每个重载的方法（或者构造函数）都必须有一个独一无二的参数类型列表。
最常用的地方就是构造器的重载。

public int test(){
        System.out.println("test1");
        return 1;
    }
 
    public void test(int a){
        System.out.println("test2");
    }   
 
    //以下两个参数类型顺序不同
    public String test(int a,String s){
        System.out.println("test3");
        return "returntest3";
    }   
 
    public String test(String s,int a){
        System.out.println("test4");
        return "returntest4";
    }

组合

注意点

  1. Java除了基本数据类型，其他都是引用类型。

  2. Java只有值传递。

  3. Java自带的数据结构：

  • LinkedList。相当于链表。实现了List接口，允许null元素。LinkedList可被用作堆栈（stack），队列（queue）或双向队列（deque）。

  • ArrayList。实现了可变大小的数组。

  • Vector。非常类似ArrayList，但是Vector是同步的。

  • Stack继承自Vector，实现一个后进先出的堆栈。

  • TreeMap 基于红黑树的平衡查找树。

  • HashMap 基于拉链法的散列表。
